# Meeting Prep: Briefing Quality Checklist

Reference file for self-review during briefing generation (Step 8) and as the
final confidence score in the briefing output. Every briefing must pass the
universal gates. Type-specific and multi-person gates apply when relevant.

---

## 1. Universal Quality Gates (All Meeting Types)

These apply to every briefing regardless of meeting type or attendee count.

### Outcome Clarity
- [ ] **Desired outcome stated in one sentence.** If you can't, the prep isn't focused enough. This sentence appears in the briefing header.
- [ ] **Every question/answer serves the outcome.** No "nice to know" filler. For each question, ask: "If they answer this, does it move us toward the outcome?" If no, cut it.
- [ ] **Unknowns are explicit.** The briefing lists what you DON'T know. Hidden unknowns are the most dangerous kind.

### Research Integrity
- [ ] **Nothing is fabricated.** Every claim is sourced (URL, prior notes) or flagged as inference. Use "likely" or "inferred from..." for educated guesses.
- [ ] **Research is current.** Company news older than 90 days is flagged as potentially stale. Role/title verified against LinkedIn within 30 days.
- [ ] **Prior context cited with evidence.** "3 prior interactions confirm..." vs. "1 observation suggests..." -- confidence scales with evidence.

### Handoff Test
- [ ] **Someone else could run this meeting.** The briefing is self-contained: context, stakes, questions, objection responses, desired outcome. No tribal knowledge assumed.
- [ ] **Worst moment prepared for.** The hardest question, the likely objection, the awkward silence -- at least one is anticipated with a response strategy.

### Structural Completeness
- [ ] **Sources section populated.** Every URL used during research is listed. No orphan claims.
- [ ] **Deliverables block present.** File path and confidence score shown.
- [ ] **Print-friendly quick reference included.** Scannable one-pager for the 2 minutes before the call.

## 2. Meeting Type Quality Gates

Identify the meeting type in Step 1. Apply the universal gates PLUS the
type-specific gates below. If the meeting doesn't fit a type cleanly, pick
the closest one and note the deviation.

### DISCOVERY / DEMO
*You're learning about them. They may be seeing the product.*

- [ ] **Pain hypothesis formed before questions written.** Questions validate or invalidate the hypothesis, not fish blindly.
- [ ] **Competitive landscape mapped.** What tools they use today, what those tools do well, where the gap is. Named tools, not "current solutions."
- [ ] **Entity orbit map completed.** Person's full professional background screened for fit, not just current company.
- [ ] **Questions are informed by research.** "What stood out?" not "Would you like to see a demo?"
- [ ] **Next step is specific.** "Book a 15-minute technical deep dive" not "Let's stay in touch."
- [ ] **No more than 12-15 questions.** Overloading signals lack of focus. Must-ask vs. if-time-allows clearly marked.

### FOLLOW-UP SALE
*You've met before. You're advancing the deal.*

- [ ] **Prior context loaded and visible.** What was discussed, promised, and decided in previous meetings. Section 1.5 populated.
- [ ] **Open threads addressed.** Every commitment from prior calls has a status: delivered, pending, dropped. Questions reference these specifically.
- [ ] **No repeated questions.** Cross-check against prior notes -- if you asked it before and got an answer, don't ask again.
- [ ] **Stakeholder map updated.** New people mentioned in prior calls added. Authority/influence assessment refined.
- [ ] **Deal momentum assessed.** Accelerating, stalling, or stuck? Evidence from meeting cadence, response times, engagement level.
- [ ] **Blocker identification.** What's preventing the next stage? Technical validation? Budget approval? Champion alignment? Questions target the blocker.

### INVESTOR MEETING
*You're being evaluated. The power dynamic is reversed.*

- [ ] **Narrative arc prepared.** Problem -> insight -> solution -> traction -> vision. Not a feature tour.
- [ ] **Metrics ready.** Revenue, growth rate, burn, runway, unit economics, pipeline -- whichever are relevant. Know your numbers cold.
- [ ] **Hard questions anticipated.** "Why hasn't this been solved before?" / "What happens if [competitor] builds this?" / "Why now?" -- at least 5 prepared with honest answers.
- [ ] **Investor background researched.** Their portfolio, thesis, check size, stage preference. Which portfolio companies overlap or conflict?
- [ ] **Ask is specific.** Amount, terms, timeline, use of funds. Vague asks signal lack of preparation.
- [ ] **Questions FOR the investor prepared.** "What would make you pass on this?" / "How do you typically help portfolio companies in our space?" -- shows confidence and evaluates fit.
- [ ] **Red flags pre-addressed.** If your metrics have a weakness (churn, CAC, small market), address it proactively rather than waiting to be caught.

### ADVISORY / MENTOR
*You're extracting strategic insight from someone generous with their time.*

- [ ] **Context dump provided upfront.** The advisor can't help without context. Prepare a 2-minute verbal summary or a pre-read document.
- [ ] **Questions are specific, not open-ended.** "Should we target [market A] or [market B] first, given [these signals]?" not "What market should we go after?"
- [ ] **Decision or dilemma framed.** What are you stuck on? What are the options? What's your current leaning? Let the advisor react, not invent from scratch.
- [ ] **Advisor's relevant experience identified.** What in their background makes them uniquely positioned to answer THIS question? Reference it.
- [ ] **Actionable takeaway target set.** "I want to leave with a decision on X" or "I need their read on whether Y is viable." Not just "catch up."
- [ ] **No selling.** You're not pitching. If you catch yourself explaining features, the prep is wrong.

### PARTNERSHIP
*Exploring mutual value. Both sides need to win.*

- [ ] **Their value chain mapped.** What do they sell, to whom, through what channels? Where does your product fit in their offering?
- [ ] **Mutual value articulated.** Not "why they should use us" but "why this makes both businesses stronger." Two-sided value prop.
- [ ] **Integration points identified.** Technical (API, data), commercial (co-sell, referral, white-label), strategic (market access, credibility).
- [ ] **Deal structure options prepared.** Revenue share, referral fee, licensing, co-development -- know what models exist in this space.
- [ ] **Their alternatives researched.** Who else could they partner with? Why you over them?
- [ ] **Champion identified.** Who internally is pushing for this partnership? What do they need to sell it to their leadership?

### CUSTOMER SUCCESS / RENEWAL
*Retention, expansion, or risk detection.*

- [ ] **Usage data reviewed.** If available: adoption metrics, feature usage, support ticket history. If not: note as a gap.
- [ ] **Health signals assessed.** Engaged champion? Executive sponsor still in role? Renewal date approaching? Competitors circling?
- [ ] **Expansion opportunity mapped.** Other teams, departments, use cases, geographies. Which have been mentioned in prior calls?
- [ ] **Risk signals flagged.** Declining usage, champion departure, budget pressure, competitor evaluation. Each with a response strategy.
- [ ] **Value delivered quantified.** What outcomes have they achieved since deployment? Numbers they can take to their leadership.
- [ ] **Renewal/expansion ask prepared.** Specific, with justification. Not "would you like to expand?" but "Given [outcome], here's what [capability] would do for [team]."

### INTERNAL (BOARD / TEAM ALIGNMENT)
*Driving a decision or building consensus.*

- [ ] **Decision to be made is explicit.** "We need to decide X" -- stated in the briefing header.
- [ ] **Options presented with trade-offs.** Not "here's what I think" but "here are the options, here's what I recommend and why."
- [ ] **Stakeholder positions pre-mapped.** Who supports what? Where's the disagreement? What information would change minds?
- [ ] **Pre-read materials prepared.** If there's a doc/deck/data, it's been sent in advance. The meeting is for discussion, not presentation.
- [ ] **Success criteria defined.** What does a good outcome look like? Unanimous agreement? Majority? Leader's call?

### HIRING / INTERVIEW
*Assessing fit in either direction.*

- [ ] **Role requirements clear.** Must-haves vs. nice-to-haves. Technical and cultural.
- [ ] **Candidate-specific questions prepared.** Not generic interview questions. "I see you built X at Y -- how did you handle Z?" based on their actual background.
- [ ] **Red flag probes included.** Career gaps, short tenures, skill claims that need validation -- addressed directly but respectfully.
- [ ] **Sell points ready.** If you want them, what's your pitch? Why this company, this role, this stage?
- [ ] **Assessment rubric defined.** What are you scoring on? Technical depth, culture fit, ambition, communication? Explicit, not vibes.

## 3. Multi-Person Quality Gates

Apply when meeting has 2+ attendees on either side. These are additive to
the universal and type-specific gates above.

### Their Side: Multiple Attendees

- [ ] **Individual dossiers for each attendee.** Not just the primary contact. Each person gets: name, title, role in meeting, likely concerns, background summary.
- [ ] **Group dynamics mapped.** Who's the decision maker? Who influences? Who might block? Who's the champion? Document the power structure.
- [ ] **Per-person question allocation.** 10-12 questions total for the meeting, not per person. Tag each question with who should answer it: "[For VP Eng]" / "[For Procurement]".
- [ ] **Conflicting agendas anticipated.** The VP may want speed, procurement wants cost reduction, legal wants risk mitigation. Note where attendees' interests diverge.
- [ ] **Common ground identified.** What do all attendees agree on? Lead with this to build momentum before navigating disagreements.
- [ ] **Seniority-appropriate depth.** Executive attendees get strategic framing. Practitioners get operational detail. Don't bore the VP with workflow steps or overwhelm the analyst with strategy.

### Your Side: Multiple Attendees

- [ ] **Role assignment documented.** Who on your side covers what topic? "CTO handles technical deep-dive, CEO handles commercial terms." No overlapping or contradicting.
- [ ] **Handoff moments planned.** "After the demo, [CTO] will walk through the architecture" -- smooth transitions, not awkward "do you want to take this?"
- [ ] **Signal protocol agreed.** How do you communicate during the meeting? A subtle cue to wrap up a topic, redirect, or let someone else speak.
- [ ] **Unified messaging confirmed.** Your side tells the same story. Key numbers, positioning, competitive framing -- aligned before the call.

### Panel / Group Presentation

- [ ] **Per-person question prediction.** For each panelist/attendee, predict their likely question angle based on their role and background. Prepare answers, not questions.
- [ ] **Answer bank prepared.** 10-15 anticipated questions with concise answers. Organized by topic, not by person, since questions may come from unexpected sources.
- [ ] **Weak spots pre-identified.** Where will you get pushed hardest? Prepare honest, confident responses. "We don't have that yet, here's why and here's our plan" beats fumbling.

## 4. Confidence Scoring Rubric

After generating the briefing, score it honestly. The score appears in the
Quick Reference section (Section 8) and helps the user calibrate how much
additional prep they need.

### Scoring Dimensions (each 1-10)

| Dimension | 1-3 (Low) | 4-6 (Medium) | 7-9 (High) | 10 (Complete) |
|-----------|-----------|--------------|------------|---------------|
| **Research depth** | Name and title only | Company website + LinkedIn | Web research + prior notes | All sources + insider knowledge |
| **Outcome clarity** | Vague ("learn more") | Directional ("qualify fit") | Specific ("confirm budget timeline, get intro to CTO") | Specific + fallback outcome |
| **Question quality** | Generic / googlable | Role-appropriate | Demonstrates homework, probes unknowns | Each question has a purpose and follow-up |
| **Risk preparation** | No objection prep | 1-2 generic objections | Role-specific objections with responses | Objections sourced from prior interactions |
| **Attendee coverage** | Primary contact only | All attendees named | Individual dossiers + dynamics | Power structure mapped + per-person strategy |

### Overall Score Calculation

```
overall = mean(research_depth, outcome_clarity, question_quality, risk_preparation, attendee_coverage)
```

| Score | Label | What It Means | Action |
|-------|-------|---------------|--------|
| 9-10 | **Ready** | Deep preparation, high confidence | Go |
| 7-8 | **Strong** | Solid prep, minor gaps noted | Review gaps, go |
| 5-6 | **Adequate** | Functional but shallow in places | Address flagged gaps if time allows |
| 3-4 | **Thin** | Significant gaps, risk of being underprepared | Spend 15 more minutes on flagged areas |
| 1-2 | **Stub** | Minimal prep, likely insufficient | Consider rescheduling or extending prep time |

### Gap Flagging

When scoring below 7 on any dimension, the briefing must include a
**Prep Gaps** callout box listing:
- Which dimension scored low
- What specific information is missing
- Suggested action to close the gap (web search, ask user, provide prior notes)

Example:
```
Prep Gaps (Confidence: 6/10)
- Attendee coverage: 4/10 -- only primary contact researched. Two other
  attendees (Jane Smith, VP Eng; Mike Chen, Procurement) have no dossiers.
  Action: Search LinkedIn for both before the call.
- Risk preparation: 5/10 -- no prior interactions to source objections from.
  Using persona-based predictions only.
```

## 5. Type Detection Heuristics

Use these signals to auto-detect meeting type when the user doesn't specify:

| Signal | Inferred Type |
|--------|--------------|
| User says "follow-up" or "we met before" + prior notes provided | Follow-up Sale |
| First meeting with no prior context | Discovery / Demo |
| User mentions "investor", "fundraise", "pitch", "VC", "raise" | Investor |
| User mentions "advisor", "mentor", "catch up", "pick their brain" | Advisory |
| User mentions "partner", "integration", "co-sell", "channel" | Partnership |
| User mentions "renewal", "QBR", "health check", "expansion" | Customer Success |
| User mentions "board", "team sync", "decision", "alignment" | Internal |
| User mentions "interview", "candidate", "hiring" | Hiring |

When ambiguous, ask: "This looks like a [type] meeting. Correct, or is it something else?"

---

## 6. How This Checklist Is Used

### During Generation (Step 8)
Before finalizing the HTML briefing, run through:
1. All universal gates
2. The detected meeting type's gates
3. Multi-person gates (if applicable)
4. Score each confidence dimension
5. Generate the overall score
6. If any dimension < 7, add the Prep Gaps callout box to the briefing

### As a Self-Review Reference
The skill reads this file at the start of Step 8 to calibrate output quality.
It defines "what good looks like" so the output is consistent across runs.

### In the Briefing Output
The confidence score and any gaps appear in:
- Section 8 (Quick Reference): `Prep Confidence: 8/10`
- Prep Gaps callout (if any dimension < 7): specific gaps + actions
